# IDEA 多项目分支快速切换

## 背景

该插件用于解决微服务开发场景中多个项目需要频繁切换分支的问题。开发者通常会在一个 IDEA
窗口中打开多个 Git 仓库，逐个切换分支耗时且容易漏掉某个服务。

## 核心能力

- 在 IDEA 右侧工具窗口展示当前工作区内的多个 Git 项目。
- 输入目标分支后，对选中的项目统一切换。
- 目标分支存在时切换到目标分支。
- 目标分支不存在时，回退到可识别的主分支，例如 `main` 或 `master`。
- 切换前检测未提交变更，并允许先使用 IDEA 搁置变更能力，失败时降级使用 Git 暂存栈。
- 使用中文明确区分 IDEA 搁置变更与 Git 暂存栈的恢复入口；两者都不自动恢复。
- 切换后不自动恢复暂存变更，由开发者手动恢复并解决可能出现的冲突。
- 支持单个项目切换和多个项目批量切换。
- 展示每个项目的切换结果、失败原因、主分支回退状态和待手动恢复提示。
- 关闭工具窗口后，当前项目会话内保留最近一次切换结果摘要；关闭 IDEA 项目后不承诺跨会话保留。
- 刷新期间展示中文加载状态、最近刷新时间，并在状态可能过期时提醒先刷新再切换。

## 初始范围

- 初始版本只处理 IDEA 当前窗口中已打开或已附加的 Git 仓库。
- 初始版本不自动拉取远端更新。
- 初始版本不自动创建缺失分支。
- 初始版本不自动恢复搁置变更或暂存栈内容。
- 批量切换允许部分项目成功、部分项目失败。

## 实施入口

- 构建配置入口：`build.gradle.kts`
- Gradle 项目入口：`settings.gradle.kts`
- 插件描述入口：`src/main/resources/META-INF/plugin.xml`
- 中文资源入口：`src/main/resources/messages/CheckBranchBundle.properties`
- 持续集成入口：`.github/workflows/ci.yml`
- 标签发布入口：`.github/workflows/release.yml`

## 当前实现进度

- 阶段 1 已完成：项目骨架、Gradle Wrapper、插件描述、中文资源入口和发布流水线已就位。
- 阶段 2 已完成：工作区仓库模型、分支切换请求模型、临时变更记录模型、切换结果模型、项目级偏好状态服务、中文通知服务、工具窗口骨架、仓库表格模型、结果面板骨架与基础测试已落地。
- 阶段 3 已完成工作区仓库扫描与刷新能力：可以基于 IDEA 内容根识别 Git 仓库，读取当前分支、主分支候选、未提交变更与阻塞状态，并将刷新结果接入工具窗口列表、空状态、最近刷新时间和中文错误提示。

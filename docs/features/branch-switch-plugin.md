# IDEA 多项目分支快速切换

## 背景

该插件用于解决微服务开发场景中多个项目需要频繁切换分支的问题。开发者通常会在一个 IDEA
窗口中打开多个 Git 仓库，逐个切换分支耗时且容易漏掉某个服务。

## 核心能力

- 在 IDEA 右侧工具窗口展示当前工作区内的多个 Git 项目。
- 输入目标分支后，对选中的项目统一切换。
- 目标分支存在时切换到目标分支。
- 目标分支不存在时，回退到可识别的主分支，例如 `main` 或 `master`。
- 切换前检测已跟踪的未提交变更，并优先使用 IDEA Shelf 搁置变更。
- Git 暂存栈能力仅保留为代码级显式兜底，不作为默认产品入口暴露。
- 使用中文明确提示 IDEA Shelf 的手动恢复入口。
- 切换后不自动恢复搁置变更，由开发者手动恢复并解决可能出现的冲突。
- 支持单个项目切换和多个项目批量切换。
- 展示每个项目的切换结果、失败原因、主分支回退状态和待手动恢复提示。
- 关闭工具窗口后，当前项目会话内保留最近一次切换结果摘要；关闭 IDEA 项目后不承诺跨会话保留。
- 刷新期间异步执行仓库状态读取，展示中文加载状态、进度条和最近刷新时间，避免卡死 IDEA 页面。

## 初始范围

- 初始版本只处理 IDEA 当前窗口中已打开或已附加的 Git 仓库。
- 初始版本不自动拉取远端更新。
- 初始版本不自动创建缺失分支。
- 初始版本不自动恢复搁置变更或暂存栈内容。
- 批量切换允许部分项目成功、部分项目失败。

## 实施入口

- 构建配置入口：`build.gradle.kts`
- Gradle 项目入口：`settings.gradle.kts`
- 插件描述入口：`src/main/resources/META-INF/plugin.xml`
- 中文资源入口：`src/main/resources/messages/CheckBranchBundle.properties`
- 持续集成入口：`.github/workflows/ci.yml`
- 标签发布入口：`.github/workflows/release.yml`

## 当前实现进度

- 阶段 1 已完成：项目骨架、Gradle Wrapper、插件描述、中文资源入口和发布流水线已就位。
- 阶段 2 已完成：工作区仓库模型、分支切换请求模型、临时变更记录模型、切换结果模型、项目级偏好状态服务、中文通知服务、工具窗口骨架、仓库表格模型、结果面板骨架与基础测试已落地。
- 阶段 3 已完成工作区仓库扫描与刷新能力：可以基于 IDEA 内容根识别 Git 仓库，读取当前分支、主分支候选、未提交变更与阻塞状态，并将刷新结果接入工具窗口列表、空状态、最近刷新时间和中文错误提示。
- 阶段 4 已完成目标分支批量切换能力：支持目标分支命中、本地主分支回退、远端分支命中、输入校验、批量执行按钮接入，以及逐仓库结果写入结果面板和表格摘要。
- 阶段 5 已完成未提交变更保护：选中仓库存在已跟踪的未提交变更时会先弹出中文选择提示，用户可选择 IDEA Shelf 搁置变更、跳过仓库或取消整批操作；Git 暂存栈实现暂时保留为代码级兜底，不作为默认入口暴露。切换成功后结果会标记“待恢复”并提示用户到 IDEA Shelf 手动恢复。
- 阶段 6 已完成单仓库精确切换与结果面板瘦身：支持只切换单个已选仓库，不影响未选仓库；结果区改为简短摘要卡片，不再展示冗长句子列表。
- 阶段 7 已完成结果反馈增强：支持成功、回退、失败、跳过等混合结果测试，通知区分普通完成与包含警告的完成场景，并在当前项目会话内保留最近一次切换结果摘要；结果卡片支持悬浮查看完整失败原因。
- 当前界面已补充“全选 / 取消全选”按钮，刷新仓库后默认全选全部可见仓库，便于直接进行批量切换。
- 当前刷新流程已改为后台任务并在界面展示进度条，降低多仓库扫描和分支预览导致的 UI 卡顿风险。

# 开源插件后续问题与建议

## 发布前建议优先处理

- [未完成] 补充 JetBrains Marketplace 所需的 `change-notes` 或独立 `CHANGELOG.md`，避免发布时只有功能描述、没有版本变更说明。
- [已完成] 明确 `verifyPlugin` 的发布门禁策略：CI 和 GitHub Release 默认调用，线上允许下载 verifier 所需 IDE；开发者本机只能通过 `localVerificationIdePath` 使用已安装 IDEA 做验证，避免下载大量 IDE。
- [未完成] 为 GitHub Release 工作流确认上传产物为 `build/distributions/*.zip`，不要误用 Marketplace 发布任务。
- [未完成] 增加发布前手动验收截图或 GIF，README 后续可以展示工具窗口、变更保护弹窗和结果卡片。
- [未完成] 明确版本号策略，例如 `0.1.0` 用于首个可用开源版本，后续按 SemVer 维护。

## 用户体验建议

- [未完成] 目标分支输入框可以补充最近使用分支下拉历史，减少重复输入。
- [未完成] 主分支候选顺序可以提供设置入口，例如允许团队配置 `develop`、`release`、`main`、`master`。
- [未完成] 结果卡片可以提供一键复制完整失败原因，方便用户发到群里或 issue。
- [未完成] 批量切换前可以增加二次确认摘要，例如显示将切换的仓库数量、有变更仓库数量、将跳过仓库数量。
- [未完成] Shelf 成功后可以在结果卡片中展示更明确的 Shelf 名称，方便用户在 IDEA Shelf 面板中定位。
- [未完成] 对“纯未跟踪文件不触发保护”的设计增加 UI 说明，避免用户误以为插件漏检。

## 安全与可靠性建议

- [未完成] 为 “checkout 会覆盖未跟踪文件” 的 Git 失败场景提供更友好的中文解释和处理建议。
- [未完成] 对多仓库批量切换增加操作审计日志，至少记录仓库、目标分支、最终分支、状态和失败原因，不记录代码内容。
- [未完成] 考虑在切换前刷新 IDEA VFS 和 ChangeListManager，降低刚编辑文件但状态未同步导致的误判风险。
- [未完成] 评估是否需要把 `fetch --prune origin` 作为可配置项；部分团队可能不希望插件执行远端刷新。
- [未完成] 对远端分支切换策略补充更多测试，例如远端存在但本地同名分支有未合并提交时的行为。

## 开源项目治理建议

- [未完成] 增加 `CONTRIBUTING.md`，说明开发环境、测试命令、代码风格、文档同步规则。
- [未完成] 增加 issue 模板，包括 Bug Report、Feature Request、Branch Switch Failure 三类。
- [未完成] 增加 PR 模板，要求说明测试结果、影响范围和文档是否同步。
- [未完成] 增加 `CHANGELOG.md`，每次发布前维护用户可读变更。
- [未完成] README 后续可以增加英文简介，方便 JetBrains Marketplace 或 GitHub 英文用户理解。
- [未完成] 如需 JetBrains Marketplace 正式发布，再单独设计更完整的插件验证矩阵；本地仍需避免下载大量 IDE。

## 技术债建议

- [未完成] 当前 Git stash 仍保留为代码级兜底能力，后续如果确认长期不需要，可以在单独版本中删除相关分支和测试，减少误解。
- [未完成] 部分 Git 命令仍由服务层直接调用，后续可评估是否进一步接入 IDEA Git4Idea 能力，但需要避免为了抽象而重构过度。
- [未完成] 为 `IdeaShelveExecutor` 增加平台集成测试或手动测试脚本说明，当前自动化测试无法真正验证 IDE Shelf UI 侧效果。
- [未完成] 表格状态和结果卡片的文案可继续抽到资源文件，减少 Java 类中的硬编码中文。
- [未完成] 后续可为多仓库状态刷新增加并发控制和超时提示，避免单个异常仓库拖慢整体刷新。

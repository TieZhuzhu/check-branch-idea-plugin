# 用户体验检查清单：IDEA 多项目分支快速切换

**用途**：验证功能规格是否已经清楚定义用户体验，足以进入实施和评审
**创建日期**：2026-06-03
**功能**：[spec.md](../spec.md)

**说明**：该清单由 `/speckit-checklist` 命令根据功能上下文和需求生成。

## 工具窗口体验

- [ ] CHK001 是否已清楚定义右侧工具窗口入口、停靠行为和固定使用预期？[清晰度，规格 §FR-001]
- [ ] CHK002 是否已完整定义仓库行字段，包括当前分支、可切换状态、未提交变更状态和最近结果？[完整性，规格 §FR-003]
- [ ] CHK003 是否已定义没有可发现 Git 仓库时的空状态需求？[缺口，规格 §US1]
- [ ] CHK004 是否已定义加载、刷新、数据过期和刷新失败等状态需求？[覆盖度，规格 §FR-017]

## 选择与分支输入

- [ ] CHK005 是否已清楚定义全选、部分选择、禁用仓库和单仓库切换的选择需求？[清晰度，规格 §FR-005]
- [ ] CHK006 是否已定义目标分支输入的校验、最近输入、空白处理和非法分支名需求？[完整性，规格 §FR-004]
- [ ] CHK007 批量切换和单仓库切换的需求是否保持一致？[一致性，规格 §US2，规格 §US4]
- [ ] CHK008 是否已清楚定义仅存在远端目标分支时的切换前体验预期？[清晰度，规格 §边界场景]

## 变更保护提示

- [ ] CHK009 是否已定义未提交变更时的三种选择提示：临时保存、跳过仓库、取消操作？[完整性，规格 §FR-011]
- [ ] CHK010 是否已清楚说明“搁置变更”和“Git 暂存栈”的用户可见差异？[清晰度，规格 §FR-012，规格 §FR-013]
- [ ] CHK011 是否已定义如何告知用户切换后不会自动恢复搁置或暂存栈变更？[覆盖度，规格 §FR-014]
- [ ] CHK012 是否已明确临时保存失败时是跳过仓库还是停止全部操作？[歧义，规格 §US3]

## 结果反馈

- [ ] CHK013 是否已定义所有结果状态的用户含义，包括成功、主分支回退、跳过、失败和待手动恢复？[完整性，规格 §FR-015]
- [ ] CHK014 是否已清楚定义部分成功且不自动回滚时的混合结果反馈？[清晰度，规格 §边界场景]
- [ ] CHK015 是否已为需要手动恢复或解决冲突的仓库定义恢复提示？[覆盖度，规格 §US5]
- [ ] CHK016 是否已定义操作期间关闭工具窗口或项目后的结果保留或展示要求？[边界场景，规格 §边界场景]

## 错误与冲突状态

- [ ] CHK017 是否已统一定义合并、变基、拣选提交和未解决冲突等阻塞状态？[一致性，规格 §FR-016]
- [ ] CHK018 对无法切换到目标分支或主分支的仓库，是否要求展示用户可理解的失败原因？[完整性，规格 §US2]
- [ ] CHK019 是否已清楚定义同时存在 `main` 和 `master` 时的选择规则及其用户展示方式？[清晰度，规格 §边界场景]
- [ ] CHK020 是否已定义仓库在刷新或切换过程中变为不可用时的需求？[缺口，规格 §边界场景]

## 可访问性与本地化

- [ ] CHK021 是否已为仓库选择、目标分支输入、刷新和切换动作定义键盘导航需求？[缺口，规格 §FR-001]
- [ ] CHK022 是否已为状态标识和结果状态定义无障碍标签或屏幕阅读器预期？[缺口，规格 §FR-003，规格 §FR-015]

- [ ] CHK026 取消、跳过、重试和临时保存失败恢复等用户体验验收场景是否完整？[覆盖度，规格 §US3，规格 §US5]

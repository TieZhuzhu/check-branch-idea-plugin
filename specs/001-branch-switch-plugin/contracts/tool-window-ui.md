# 界面契约：右侧分支切换工具窗口

## 契约范围

本契约定义右侧工具窗口对用户呈现的界面状态、操作入口和结果反馈。它用于约束实现和验收，不描述内部代码细节。

## 工具窗口

- 工具窗口名称使用中文，建议为“分支切换”。
- 工具窗口注册在 IDEA 右侧。
- 工具窗口在存在 Git 仓库的项目中可见。
- 工具窗口内容必须支持固定停靠使用。

## 顶部操作区

必须包含以下用户可见元素：

- 目标分支输入框。
- 最近目标分支选择能力。
- 刷新仓库状态按钮。
- 执行切换按钮。
- 当前选择数量提示。
- 最近刷新时间提示。

中文文案要求：

- 空输入提示使用中文。
- 非法分支名提示使用中文。
- 没有选中仓库时的禁用原因使用中文。
- 刷新进行中和状态可能过期提示使用中文。

## 仓库列表区

每个仓库必须展示：

- 仓库名称。
- 仓库根路径。
- 当前分支。
- 目标分支状态。
- 未提交变更状态。
- 可切换状态。
- 是否参与本次操作。
- 最近一次操作结果摘要。

状态文案必须区分：

- 可切换。
- 目标分支存在。
- 目标分支仅远端存在，将从远端检出。
- 将回退到主分支。
- 有未提交变更。
- 正在合并、变基、拣选提交或存在冲突。
- 不可识别 Git 仓库。

## 未提交变更提示

当选中仓库存在未提交变更时，必须让用户在执行前选择：

- 临时保存变更后继续。
- 跳过该仓库。
- 取消本次操作。

如果 IDEA 搁置变更不可用或失败，必须明确提示是否降级使用 Git 暂存栈。

用户可见说明必须明确区分：

- IDEA 搁置变更：在 IDEA 内生成搁置记录，后续由用户手动取消搁置并处理冲突。
- Git 暂存栈：生成 Git 暂存记录，后续由用户手动 `stash pop` 或 `stash apply` 并处理冲突。
- 如果搁置变更失败且 Git 暂存栈不可用或用户拒绝降级，默认跳过当前仓库并继续其他安全仓库；只有用户明确取消时才终止剩余处理。

## 结果区

每个仓库必须展示最终状态：

- 成功。
- 主分支回退。
- 跳过。
- 失败。
- 待手动恢复。

结果必须包含：

- 请求分支。
- 最终分支。
- 是否使用主分支回退。
- 目标分支来源，区分本地分支或远端分支。
- 是否产生临时变更记录。
- 用户下一步动作提示。

## 空状态与异常状态

- 未发现 Git 仓库时，必须展示中文空状态说明。
- 刷新进行中时，必须展示中文加载状态并禁用重复刷新操作。
- 当前展示状态不是最近一次刷新结果时，必须提示状态可能已过期并建议先刷新再切换。
- 刷新失败时，必须展示中文失败原因。
- 仓库在操作中消失或不可访问时，必须展示中文跳过或失败原因。
- 操作期间关闭工具窗口时，当前项目会话内必须保留最近一次切换结果；重新打开工具窗口后必须展示最近一次结果摘要；关闭 IDEA 项目后不要求跨会话保留。
- 切换失败后，用户无需重新打开工具窗口即可再次刷新并重试。

## 可访问性

- 目标分支输入、刷新、切换和仓库选择必须支持键盘访问。
- 仓库勾选框、刷新按钮、切换按钮、状态图标和结果状态必须提供可访问名称或说明，便于屏幕阅读器识别。
- 状态不能只依赖颜色表达，必须同时提供文字或图标含义。
- 重要失败和待恢复状态必须有清晰中文说明。
- 单仓库切换必须沿用与批量切换一致的输入校验、变更保护、回退规则和结果反馈。
